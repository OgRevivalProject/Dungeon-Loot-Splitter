🧙 Logic Explanation

Array: All loot items are stored in a single JavaScript array called lootItems declared at the top of script.js. Each element in the array is an object with two properties: name (a string) and value (a number). This array persists in memory across button clicks because it lives in the outer scope of the script.

Loops: A traditional for loop iterates over lootItems inside renderLoot() to build the HTML list displayed on the page. A second loop inside renderLoot() (and used by splitLoot()) accumulates the total by adding each item's value to a running sum variable.

Conditional Logic: Inside addLoot(), if statements validate that the loot name is not empty, the loot value is a valid number, and the value is not negative before pushing to the array. Inside splitLoot(), if statements check that party size is at least 1 and that the loot array is not empty before performing division. Errors are displayed inside the UI rather than using alerts.
